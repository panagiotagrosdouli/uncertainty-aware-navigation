# Contributing

Thank you for considering a contribution to `uncertainty-aware-navigation`.

This repository is research software. Contributions should improve scientific validity, reproducibility, implementation clarity, or usability.

## Development setup

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -e '.[dev]'
pre-commit install
```

## Quality checks

Run these before opening a pull request:

```bash
ruff check src tests experiments
black --check src tests experiments
pytest
```

## Research contribution rules

- Do not add benchmark numbers unless they are generated by committed scripts.
- Include the configuration, seeds, and command used to produce each result.
- Label work as Implemented, Prototype, or Planned.
- Prefer deterministic tests over visual inspection.
- Do not redistribute datasets, maps, or logs unless their license explicitly permits it.

## Code style

- Use type hints for public functions.
- Use Google-style docstrings for public modules, classes, and functions.
- Keep algorithms in reusable modules under `src/`, not only in notebooks or scripts.
- Use structured logging for experiment scripts.
- Keep configuration in YAML or typed config objects.

## Pull request expectations

A strong pull request includes:

1. a clear motivation;
2. tests or an explanation of why tests are not applicable;
3. updated documentation;
4. no fabricated or manually edited benchmark results;
5. generated artifacts only when they can be reproduced from scripts.
